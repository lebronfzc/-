# 使用案例：Codex 王炸技巧封面

这个案例展示了为什么这个 skill 采用「先做画面，再本地排版文字」的流程。

## 用户需求

用户提供了一张小红书封面参考图，并提出这些修改：

- 主标题改成：`codex 王炸技巧`
- 副标题改成：`夯爆封面 skill`
- 中间图标换成 Codex 风格图标
- 最下方文字换成：`codex design`

## 第一次结果的问题

第一次直接让图片生成模型同时处理画面和中文文字。

优点：

- 整体氛围接近参考图
- 颜色、冲击力、构图方向比较对

问题：

- 中文字体不稳定
- 容易缺字、错字、变形
- 不适合需要精准文案的小红书封面

## 第二次结果为什么更好

第二次改成两步：

1. 先做封面底图、图标、背景、层次和整体视觉。
2. 再用本地字体把中文标题和底部文字准确排上去。

这样做的好处：

- 文案不会缺字
- 主标题、副标题、底部文字都更清楚
- 字体大小、描边、阴影、位置都可控
- 更适合真正发布的小红书封面

## 推荐使用方式

直接这样说：

```text
用小红书封面创作，参考这张图。
主标题：codex 王炸技巧
副标题：夯爆封面 skill
图标换成 codex
底部：codex design
```

如果标题比较长，建议拆成主标题和副标题，不要全部塞进一行。
